## 2. 语音对话架构详解

### 2.1 数据流

```
用户说话
    │
    ▼
[SimpleRecorder] 16kHz PCM Int16
    │
    ▼
[ViberSocket] Base64 编码 → 发送
    │
    ▼
[VoiceHandler] 接收 → Buffer 解码
    │
    ▼
[ASR Session] 火山引擎 WebSocket
    │
    ├──→ [Interim] ────────────────┐
    │                               ▼
    │                   [前端] 停止 TTS 播放
    │                               │
    └──→ [Final] ──────────────────┘
                    │
                    ▼
            [VoiceOrchestrator]
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
[ChatService]              [Interrupt]
(kimi-cli 流式)            (停止当前 LLM)
        │                       │
        ▼                       │
[LLM Token 流]                │
        │                       │
        ▼                       │
[TTS Trigger?] ───────────────┘
(20字符/标点)
        │
        ├──→ [Thinking] → 立即 TTS
        │
        └──→ [Response] → 流式 TTS
                    │
                    ▼
            [Text Cleaning]
            (kimi-cli --final-message-only)
                    │
                    ▼
            [Volcano TTS]
                    │
                    ▼
            [Speaker:play]
                    │
                    ▼
        [SpeakerController]
        (前端音频队列播放)
```

### 2.2 打断机制

| 触发场景 | 处理流程 |
|---------|----------|
| **ASR Interim** (用户开始说话) | 立即停止前端 TTS 播放 (streamInfo.ttsStopped = true) |
| **ASR Final** (识别完成) | 1. voiceOrchestrator.interrupt()<br>2. ChatService.stopGeneration()<br>3. 发送 speaker:stop |
| **手动打断** | 同上 |
| **断开连接** | ViberSocketManager.onDisconnect() → voiceOrchestrator.endDialog() |

### 2.3 关键配置

```javascript
// TTS 触发阈值
const TTS_TRIGGER_LENGTH = 20;  // 字符数
const TTS_TRIGGER_PUNCT = /[。！？.!?\n]/;  // 标点符号

// 音频参数
const AUDIO_SAMPLE_RATE = 16000;  // 16kHz
const AUDIO_FRAME_SIZE = 4096;    // 样本数

// 超时配置
const CLEAN_TEXT_TIMEOUT = 3000;  // 文本清洗 3s 超时
const ASR_CONNECT_TIMEOUT = 5000; // ASR 连接 5s 超时
```

## 4. 关键技术决策

### 4.1 为什么使用服务端 ASR/TTS？

| 方案 | 延迟 | 成本 | 质量 | 选择 |
|------|------|------|------|------|
| 浏览器 Web Speech API | ~500ms | 免费 | 中 | ❌ |
| 本地 Whisper | 1-3s | 免费 | 高 | ❌ |
| **火山引擎** | **~300ms** | **低** | **高** | **✅** |

**原因**: 火山引擎提供生产级的实时语音服务，延迟低、准确率高，适合"打电话"式体验。

### 4.2 为什么使用子进程调用 Kimi CLI？

- 确保使用用户的 **订阅计划** (而非 Token 计费)
- 支持 **工具调用** (文件操作、搜索等)
- **流式输出** (`--output-format=stream-json`)

### 4.3 为什么使用统一 WebSocket？

- 单连接管理，减少资源消耗
- 统一消息格式，简化前后端通信
- 便于扩展新功能
